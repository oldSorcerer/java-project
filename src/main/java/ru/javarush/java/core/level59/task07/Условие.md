
### Чтение чанка файла через MappedByteBuffer

Представьте, что вы — картограф данных, и перед вами лежит гигантский, неизведанный "континент" информации — огромный файл. Вам не нужно исследовать его весь сразу, но вы хотите быстро заглянуть в определённый, очень важный "регион", например, второй по счёту. Загружать весь континент в память — слишком долго и ресурсоёмко.

Ваша задача — реализовать программу, которая использует магию MappedByteBuffer для создания прямого "окна" из вашей оперативной памяти во второй "регион" этого файла. Пусть каждый такой регион имеет размер 4 мегабайта. Как только это "окно" будет открыто, вам нужно будет аккуратно "прочитать" и вывести на экран первые 100 символов этого региона, чтобы получить первое представление о том, что там находится. Если регион окажется меньше 100 символов, просто выведите всё, что смогли прочитать. Это позволит вам эффективно исследовать огромные массивы данных, не теряя времени на избыточные операции.

#### Требования:
1. Программа должна использовать класс MappedByteBuffer для отображения части файла в память.
2. Для работы с MappedByteBuffer необходимо открыть файл с помощью FileChannel.
3. В программе должен быть реализован переход ко второму региону файла, начиная с позиции 4 мегабайта (4 * 1024 * 1024 байта).
4. Размер отображаемого региона должен составлять ровно 4 мегабайта.
5. Из второго региона файла необходимо прочитать первые 100 символов (или меньше, если регион меньше 100 символов).
6. Прочитанные символы должны быть корректно выведены на экран в том виде, в каком они содержатся в файле.
7. Если во втором регионе содержится меньше 100 символов, программа должна вывести только доступные символы, не выходя за границы отображённого региона.
8. Все используемые ресурсы (например, FileChannel, RandomAccessFile) должны быть корректно закрыты после завершения работы программы.
