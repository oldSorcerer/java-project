
### Использование poison pill для завершения работы consumer

Представьте, что вы управляете производственной линией на фабрике. У вас есть автоматический "поставщик" (producer), который отправляет сырьё, и "переработчик" (consumer), который это сырьё обрабатывает. Важно, чтобы переработчик знал, когда производство завершено и больше не будет новых поставок, чтобы он мог аккуратно завершить свою работу.

Ваша задача — создать "складской конвейер" — ArrayBlockingQueue<String> вместимостью в 5 единиц сырья. В главном цехе (основном потоке) ваш "поставщик" загружает на конвейер две партии сырья: "сырье_А" и "сырье_Б". Но самое главное — после них он помещает на конвейер специальный, уникальный "сигнал о завершении производства", который вы назовёте __КОНЕЦ_СМЕНЫ__.

В отдельном рабочем блоке (отдельном потоке) реализуйте вашего "переработчика". Он должен непрерывно забирать сырьё с конвейера (take()) и сообщать, что он обработал ("Обработано: [партия_сырья]"). Он будет продолжать работать до тех пор, пока не получит тот самый "сигнал о завершении производства". Как только он его увидит, пусть он объявляет "Смена завершена, прекращаю работу!" и аккуратно останавливается. Это обеспечивает эффективное и безопасное завершение рабочего процесса.

#### Требования:
1. В программе должен быть создан объект ArrayBlockingQueue<String> с вместимостью 5 элементов.
2. В основной поток (producer) должен помещать на конвейер две строки: "сырье_А" и "сырье_Б", используя методы очереди.
3. После партий сырья producer обязан добавить на конвейер специальную строку "__КОНЕЦ_СМЕНЫ__" для оповещения consumer о завершении работы.
4. Потребитель (consumer) должен быть реализован в отдельном потоке и забирать элементы из очереди методом take().
5. Consumer должен выводить сообщение "Обработано: [партия_сырья]" для каждой принятой партии, кроме сигнала "__КОНЕЦ_СМЕНЫ__".
6. Consumer обязан завершить свою работу, как только получит строку "__КОНЕЦ_СМЕНЫ__", и вывести сообщение "Смена завершена, прекращаю работу!".
7. После получения сигнала о завершении работы потребительский поток должен завершиться корректно.
