
### Конвейер из producer и двух consumers

Представьте, что вы — диспетчер в высокотехнологичной службе доставки, где беспилотные дроны разносят посылки. У вас есть центральный "склад" — ArrayBlockingQueue<String> вместимостью в 10 пакетов. Ваша задача — настроить систему так, чтобы один "отправитель" (producer) мог отправлять посылки, а два "дрона-доставщика" (consumers) забирали их и доставляли, при этом зная, когда пора возвращаться на базу.

В отдельном центре управления (потоке) ваш "отправитель" начинает свою работу. Он последовательно загружает на склад пять важных пакетов: "Посылка_Альфа", "Посылка_Браво", "Посылка_Чарли", "Посылка_Дельта" и "Посылка_Эхо". После этого, чтобы сигнализировать о завершении рабочего дня для каждого дрона, он аккуратно помещает на склад два специальных "сигнала возврата на базу", которые вы назовёте __БАЗА__. По одному такому сигналу для каждого дрона!

Затем вы запускаете в небо двух ваших "дронов-доставщиков". Каждый из них находится в своём отдельном полёте (потоке). Они будут непрерывно забирать посылки со склада (take()), сообщая: "Дрон [номер дрона] доставил: [название посылки]". Как только дрон получает "сигнал возврата на базу" (__БАЗА__), он должен сообщить: "Дрон [номер дрона] получил сигнал, возвращаюсь на базу!" и плавно завершить свою миссию.

Ключевая цель — убедиться, что все пять важных посылок будут обработаны одним или обоими дронами, и что оба дрона-доставщика чётко поймут, когда их работа завершена, и вернутся на базу после получения своего персонального "сигнала". Это демонстрация идеальной координации в вашей системе доставки!

#### Требования:
1. В программе должна использоваться ArrayBlockingQueue<String> с размером очереди, равным 10.
2. Producer должен работать в отдельном потоке и последовательно помещать в очередь пять строк: "Посылка_Альфа", "Посылка_Браво", "Посылка_Чарли", "Посылка_Дельта", "Посылка_Эхо".
3. Producer после отправки всех посылок должен добавить в очередь два специальных элемента "__БАЗА__" (по одному для каждого consumer).
4. Должно быть создано два потока-consumer, каждый из которых извлекает элементы из очереди с помощью метода take().
5. Каждый consumer при получении обычной посылки должен выводить сообщение: "Дрон [номер дрона] доставил: [название посылки]".
6. Если consumer получает строку "__БАЗА__", он должен вывести сообщение: "Дрон [номер дрона] получил сигнал, возвращаюсь на базу!" и завершить свою работу.
7. Все пять посылок должны быть обработаны (доставлены) одним или обоими дронами до получения ими сигнала "__БАЗА__".
8. Программа должна корректно завершать все потоки после обработки всех посылок и получения обоими consumer-ами своих сигналов "__БАЗА__".
