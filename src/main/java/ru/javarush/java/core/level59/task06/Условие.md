
### Параллельный поиск файлов по дате изменения

Вы — высококлассный цифровой детектив, и вам поручено найти "свежие следы" в огромном, запутанном цифровом архиве. Ваша цель — обнаружить все файлы, которые были изменены за последние семь дней, в главном архиве evidence_vault и всех его бесчисленных подпапках. Вручную это займёт вечность, но у вас есть передовые инструменты для расследования!

Ваша задача — создать программу, которая использует мощь Files.walk() для глубокого сканирования всего архива. Но чтобы ускорить процесс, вы задействуете параллельный стрим (parallel()), позволяя вашей программе работать как команда высокоэффективных цифровых криминалистов, одновременно прочёсывающих разные части архива. Программа должна сосредоточиться только на файлах, которые соответствуют вашему временному критерию.

В итоге, ваша программа должна представить вам чёткий список: пути ко всем найденным "уликам" — файлам, изменённым за последнюю неделю. Это позволит вам быстро сосредоточиться на самых важных и актуальных доказательствах.

#### Требования:
1. Программа должна использовать метод Files.walk() для рекурсивного обхода всех файлов и папок внутри директории evidence_vault.
2. Результирующий Stream, полученный из Files.walk(), должен быть преобразован в параллельный с помощью метода parallel() для ускорения обработки.
3. Программа должна фильтровать поток так, чтобы обрабатывались только файлы, а не директории.
4. Для каждого файла программа должна проверять дату последнего изменения и выбирать только те файлы, которые были изменены за последние 7 дней относительно текущей даты.
5. Программа должна собирать пути (Path) ко всем найденным файлам, соответствующим временному критерию, в итоговый список.
6. Программа должна вывести итоговый список путей ко всем найденным файлам на экран.
