
### Подсчёт общего размера файлов в каталоге с помощью ForkJoin

Представьте, что вы — хранитель огромного, многоуровневого цифрового архива на старинном сервере, который содержит миллионы документов, изображений и видео, заботливо разложенных по бесчисленным вложенным папкам. Вам срочно нужно узнать, какой общий объём (в байтах) занимают все эти сокровища, хранящиеся в главной папке ancient_archive и всех её подземельях. Пройтись по каждой папке вручную — это самоубийство!

К счастью, у вас есть доступ к технологии ForkJoin, которая позволяет распараллелить такую колоссальную работу. Ваша задача — создать особого "исследователя архива", класс, который будет наследоваться от RecursiveTask<Long>. Этот исследователь должен быть способен заходить в каждую папку, подсчитывать размер находящихся там файлов, и, если папка содержит ещё подпапки, делегировать их исследование своим "младшим" коллегам (другим задачам). В итоге он должен собрать воедино все результаты и вернуть общий размер в байтах.

В главном командном центре (main методе) вам предстоит активировать "пульт управления" для вашей команды исследователей — ForkJoinPool. Запустите вашу основную задачу по подсчёту размера архива и, когда вся команда завершит свою работу, с гордостью выведите на экран общую сумму в байтах. Это позволит вам, наконец, узнать истинный масштаб вашего цифрового наследия!

#### Требования:
1. Класс-исследователь архива должен наследоваться от RecursiveTask<Long> для возможности рекурсивной обработки папок и файлов с возвратом результата типа Long.
2. Класс-исследователь должен принимать в качестве параметра путь к обрабатываемой директории (например, File или Path).
3. Выполняя свою задачу, исследователь должен просматривать все элементы в переданной директории: если элемент — файл, учитывать его размер, если элемент — папка, создавать для неё новую подзадачу.
4. Для каждой обнаруженной подпапки должен создаваться новый экземпляр исследователя (RecursiveTask), который рекурсивно подсчитывает размер файлов внутри этой подпапки.
5. После завершения всех подзадач результаты (размеры файлов и подпапок) должны быть суммированы для получения общего размера текущей директории.
6. В методе main задача по подсчёту размера архива должна запускаться с помощью ForkJoinPool.
7. После запуска основной задачи main должен дождаться завершения вычислений и вывести на экран общий размер архива в байтах.
8. Класс-исследователь должен корректно обрабатывать пустые папки и возможные ошибки доступа к файлам (например, пропускать недоступные файлы, чтобы не прерывать выполнение задачи).
