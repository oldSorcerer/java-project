
### Научная лаборатория: Документирование потенциальных сбоев в данных

Вы разрабатываете систему для ввода результатов научных измерений. Некоторые измерения представляют собой числа, но иногда учёные могут случайно ввести текст вместо цифр. Вам, как разработчику, важно не только обрабатывать такие ситуации, но и ясно обозначить для других программистов, какие "подводные камни" могут встретиться при использовании вашего кода.

Создайте метод, назовите его parseMeasurementValue, который будет принимать на вход строку с данными измерения. Внутри этого метода используйте стандартный механизм для преобразования строки в целое число. Если строка не может быть преобразована в число (например, содержит буквы), это вызовет NumberFormatException.

Самое важное: оформите Javadoc-комментарии для вашего метода parseMeasurementValue. В этих комментариях, используя специальную аннотацию @throws, явно укажите, что данный метод может "выбросить" NumberFormatException, если входная строка не является корректным числом.

В главной части вашей программы вызовите parseMeasurementValue, передав ему строку "123abc", которая заведомо не является числом. Если возникнет исключение (что и произойдёт), перехватите его и выведите на экран сообщение "Ошибка преобразования числа: введены некорректные данные", чтобы пользователь понял, почему его ввод не был принят.

#### Требования:
1. Необходимо создать метод с именем parseMeasurementValue, который принимает строковый параметр.
2. Внутри метода parseMeasurementValue должна использоваться стандартная операция преобразования строки в целое число (например, Integer.parseInt).
3. Для метода parseMeasurementValue необходимо оформить Javadoc-комментарий с аннотацией @throws, в котором указано, что метод может выбросить NumberFormatException при некорректном формате строки.
4. В основной части программы вызвать метод parseMeasurementValue с аргументом "123abc" и обработать возможное исключение NumberFormatException с помощью блока try-catch.
5. В случае возникновения NumberFormatException в блоке catch вывести на экран сообщение: "Ошибка преобразования числа: введены некорректные данные".
