
### Страж ворот игры: Проверка игрового счёта

Вы разрабатываете систему подсчёта очков для вашей новой увлекательной игры. Но есть одно важное правило: счёт игрока не может быть "слишком мал", иначе это может испортить весь игровой процесс. Вам нужна особая проверка, которая "крикнет" об ошибке, если кто-то попытается ввести недопустимое значение.

Ваша миссия — создать специальный метод, назовите его checkPlayerScore. Этот метод должен принимать на вход целое число, представляющее предполагаемый счёт игрока. Если это число меньше 10, метод должен "выбросить" ваше собственное, специально созданное исключение, например, ScoreTooLowException, которое будет наследником обычного Exception. Это исключение будет сигнализировать о том, что "Счёт игрока слишком низкий!". Если же счёт в порядке (10 или больше), метод просто спокойно продолжит свою работу.

Таким образом, вы создадите надёжный страж ворот, который будет гарантировать, что счёт игрока всегда будет значимым и ваша игра не столкнётся с нелепыми значениями.

#### Требования:
1. Необходимо создать собственный класс исключения с именем ScoreTooLowException, который должен наследоваться от класса Exception.
2. Класс ScoreTooLowException должен иметь конструктор, принимающий строку с сообщением об ошибке.
3. В программе должен быть реализован метод checkPlayerScore, принимающий один параметр типа int.
4. Метод checkPlayerScore должен проверять, что переданное значение меньше 10.
5. Если переданное значение меньше 10, метод checkPlayerScore должен выбрасывать исключение ScoreTooLowException с сообщением "Счёт игрока слишком низкий!".
6. Если переданное значение 10 или больше, метод checkPlayerScore не должен выбрасывать никаких исключений и должен завершаться корректно.
