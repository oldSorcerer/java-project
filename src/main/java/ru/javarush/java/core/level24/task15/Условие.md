
### Секретный архив: Приоритет доступа к конфиденциальным данным

Вы создаёте систему управления доступом к важным документам в секретном архиве. Некоторые документы особенно конфиденциальны, и если к ним не удаётся получить доступ, это нужно обработать по-особому. Однако есть и более общие проблемы с доступом ко всем документам. Нужно быть очень точным в том, как вы обрабатываете эти ситуации.

Создайте программу, в которой блок, отвечающий за попытку доступа к документу try, может столкнуться как с ситуацией, когда конкретный секретный документ не найден (FileNotFoundException), так и с более общей проблемой ввода-вывода при работе с архивом (IOException).

Ваша задача — расположить "планы спасения" (catch-блоки) в правильном порядке: сначала должен идти блок для FileNotFoundException, а затем для более общего IOException. Важно понимать, что если вы попытаетесь сначала поймать общий тип (IOException), то более специфичный (FileNotFoundException) уже не "доберется" до своего обработчика, и программа не скомпилируется.

При рабочем варианте, если конкретный файл не найден, программа должна вывести "Секретный документ не найден!", а при общей ошибке ввода-вывода — "Ошибка доступа к архиву!".

#### Требования:
1. Блок catch для FileNotFoundException должен идти перед блоком catch для IOException.
2. Если возникает исключение FileNotFoundException, программа должна вывести на экран "Секретный документ не найден!".
3. Если возникает исключение IOException (не являющееся FileNotFoundException), программа должна вывести на экран "Ошибка доступа к архиву!".
4. В программе должен быть блок try, в котором осуществляется попытка доступа к документу, и соответствующие блоки catch для обработки исключений.
5. Программа должна компилироваться без ошибок, связанных с порядком catch-блоков для исключений разной специфичности.
