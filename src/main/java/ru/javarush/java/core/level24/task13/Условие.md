
### Финансовый отчёт: Точная диагностика проблем с данными

Вы разрабатываете критически важную систему для финансового отдела, которая должна обрабатывать ежедневные транзакции из файла. Если что-то пойдёт не так, бухгалтеры должны получить максимально чёткую информацию о том, что случилось: файл не найден, возникли общие проблемы при чтении, или же данные внутри файла оказались "испорченными".

Ваша задача — написать программу, которая попытается открыть файл под названием "daily_transactions.txt". Затем она попробует прочитать первую строку из этого файла и преобразовать её в целое число, которое могло бы быть суммой транзакций.

Чтобы чётко сообщать, что именно пошло не так, используйте несколько отдельных catch-блоков:

Если файл "daily_transactions.txt" не будет найден (FileNotFoundException), ваша программа должна вывести сообщение: "Файл транзакций не найден!".
Если произойдёт любая другая ошибка при попытке прочитать файл (IOException), выведите: "Ошибка чтения данных из файла!".
Если же первая строка файла окажется не числом, а каким-то бессмысленным текстом (NumberFormatException), программа должна сообщить: "В файле обнаружены некорректные финансовые данные!".
Таким образом, вы создадите надёжный инструмент для финансового отдела, который позволит им точно знать, как реагировать на каждую конкретную проблему с отчётом.

#### Требования:
1. Программа должна пытаться открыть файл с именем "daily_transactions.txt" для чтения.
2. Программа должна считать первую строку из файла "daily_transactions.txt".
3. Программа должна попытаться преобразовать первую строку файла в целое число.
4. Если файл "daily_transactions.txt" не найден, программа должна вывести сообщение: "Файл транзакций не найден!".
5. Если при чтении файла возникает любая другая IOException, программа должна вывести сообщение: "Ошибка чтения данных из файла!".
6. Если первая строка файла не может быть преобразована в число (NumberFormatException), программа должна вывести сообщение: "В файле обнаружены некорректные финансовые данные!".
7. Для каждого типа ошибки (FileNotFoundException, IOException, NumberFormatException) должны быть реализованы отдельные блоки catch с соответствующими сообщениями.
