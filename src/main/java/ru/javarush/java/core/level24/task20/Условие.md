
### Цифровой архивариус: Безопасное копирование древних рукописей

Вы — цифровой архивариус, которому поручено переносить древние рукописи из одного хранилища в другое. Каждая строка текста — это ценность, и вам крайне важно сделать это аккуратно, гарантируя, что и источник, и пункт назначения будут безопасно обработаны и закрыты после операции, даже если возникнут проблемы.

Ваша задача — написать программу, которая будет копировать содержимое файла "ancient_manuscript.txt" в новый файл "digital_archive.txt", построчно перенося каждый абзац. Используйте для этого мощную конструкцию try-with-resources, которая позволит вам одновременно открыть сразу два "инструмента": один для чтения (BufferedReader) исходного манускрипта и другой для записи (BufferedWriter) в новый архив. Это обеспечит, что оба "инструмента" будут автоматически закрыты, как только работа с ними завершится, независимо от успеха или неудачи операции.

После завершения копирования в файле "digital_archive.txt" должно оказаться точно такое же содержимое, как и в "ancient_manuscript.txt". Если в процессе копирования возникнет какая-либо ошибка (например, с доступом к файлам), ваша программа должна перехватить её и вывести сообщение: "Ошибка копирования рукописи!". Таким образом, вы обеспечите надёжность и безопасность в своей работе архивариуса.

#### Требования:
1. Программа должна использовать конструкцию try-with-resources для открытия и автоматического закрытия BufferedReader и BufferedWriter.
2. Программа должна читать исходный файл "ancient_manuscript.txt" построчно и записывать каждую строку в файл "digital_archive.txt".
3. В результате работы программы содержимое файла "digital_archive.txt" должно полностью совпадать с содержимым "ancient_manuscript.txt".
4. Если при копировании возникнет исключение (например, IOException), программа должна перехватить его и вывести сообщение: "Ошибка копирования рукописи!".
5. BufferedReader должен использоваться для чтения, а BufferedWriter — для записи, оба должны быть корректно связаны с соответствующими файлами.
