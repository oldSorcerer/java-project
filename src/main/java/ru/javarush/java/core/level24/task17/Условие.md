
### Новостной агрегатор: Автоматическое чтение заголовка статьи

Вы пишете программу для автоматического новостного агрегатора, которая должна быстро считывать заголовки статей из файлов. Крайне важно, чтобы после прочтения файла все ресурсы были аккуратно закрыты, даже если произошла ошибка.

Ваша задача — разработать механизм, который будет читать первую строку файла под названием "article_text.txt" и выводить её на экран. Используйте для этого специальную конструкцию try-with-resources. Это гарантирует, что даже если файл не найден или произошла другая ошибка при чтении, используемые ресурсы будут автоматически и корректно закрыты.

Если по какой-либо причине (например, файл не существует или есть проблемы с доступом) произойдёт ошибка при чтении файла, ваша программа должна перехватить эту ситуацию и вывести сообщение "Ошибка чтения статьи!". Таким образом, ваш агрегатор будет надёжно работать, обеспечивая чистоту ресурсов.

#### Требования:
1. В программе должен быть использован блок try-with-resources для автоматического закрытия ресурсов при работе с файлом.
2. Программа должна прочитать только первую строку из файла "article_text.txt".
3. Прочитанная первая строка должна быть выведена на экран.
4. В случае возникновения любой ошибки при чтении файла (например, файл не найден или нет доступа), программа должна перехватить исключение.
5. Если при чтении файла возникла ошибка, программа должна вывести сообщение "Ошибка чтения статьи!" на экран.
