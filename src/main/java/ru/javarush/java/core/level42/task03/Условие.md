
### Маркерный интерфейс Serializable

Представьте, что вы — главный менеджер склада в волшебном магазине. У вас есть множество уникальных товаров, и вам нужно сделать их "переносимыми" или "замороженными", чтобы их можно было легко сохранить или переместить между измерениями, не теряя их магических свойств.

Создайте класс под названием Product, который будет описывать ваш товар, храня его название в переменной itemName (типа String) и цену в переменной itemPrice (типа int).

Ваша задача — убедиться, что объекты этого класса можно "упаковать" для дальнейшего хранения или передачи стандартными магическими средствами Java. Для этого просто укажите необходимое "клеймо" в объявлении класса, не добавляя никаких дополнительных заклинаний или методов.

#### Требования:
1. Необходимо создать класс с именем Product.
2. Класс Product должен содержать два поля: itemName типа String и itemPrice типа int.
3. Класс Product должен реализовывать интерфейс Serializable.
4. В классе Product не должно быть собственных методов для сериализации или десериализации (writeObject/readObject и др.).
5. Объекты класса Product должны быть сериализуемыми стандартными средствами Java без дополнительной настройки.
