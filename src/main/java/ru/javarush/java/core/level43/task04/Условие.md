
### Секреты Зарплаты: Временное Исчезновение и Восстановление По Умолчанию

Вы – HR-менеджер в инновационной компании "Цифровые Мастера", и вам нужно создать систему управления данными сотрудников. Каждый сотрудник (Employee) имеет employeeName, но его employeeSalary (зарплата) является конфиденциальной информацией и не должна быть частью постоянного файла записи, если только это не абсолютно необходимо для расчетов в реальном времени. Класс Employee должен быть Serializable.

Для достижения этой цели сделайте поле employeeSalary transient. Это означает, что оно не будет сохраняться при сериализации. Но что делать, если после загрузки записи сотрудника нам все-таки нужно установить какое-то значение зарплаты, например, для расчетов? Реализуйте приватный метод readObject(ObjectInputStream in), который будет вызываться после стандартной десериализации. В этом методе вызовите стандартную десериализацию, а затем, словно по волшебству, установите employeeSalary в значение 1000.

В основном методе вашей программы создайте объект Employee, установите его employeeSalary в 5000, затем "сохраните" его (сериализуйте) и "загрузите" обратно (десериализуйте). Наконец, выведите на экран employeeName и employeeSalary восстановленного объекта. Ваша проверка будет успешной, если employeeSalary будет равна 1000, несмотря на то, что изначально она была 5000, демонстрируя, что конфиденциальные данные не сохранялись напрямую, а были восстановлены по правилу по умолчанию.

#### Требования:
1. Класс Employee обязан реализовывать интерфейс Serializable для поддержки сериализации объектов.
2. Поле employeeSalary должно быть объявлено с модификатором transient, чтобы оно не сохранялось при сериализации.
3. В классе Employee должен быть реализован приватный метод readObject(ObjectInputStream in), который вызывается при десериализации.
4. В методе readObject(ObjectInputStream in) необходимо вызвать in.defaultReadObject() для выполнения стандартной десериализации остальных полей.
5. После стандартной десериализации в методе readObject поле employeeSalary должно быть установлено в значение 1000.
6. В основной части программы должен быть создан объект Employee, его поле employeeSalary установлено в 5000, после чего объект сериализуется и десериализуется обратно.
7. После десериализации необходимо вывести на экран значения employeeName и employeeSalary восстановленного объекта, при этом значение employeeSalary должно быть равно 1000.
