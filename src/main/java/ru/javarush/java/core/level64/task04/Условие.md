
### Изменение сборщика мусора и настройка паузы – Перфекционист Настройки Производительности ⚙️

Вы – высококлассный инженер по производительности, и вам поручено настроить критически важное серверное приложение (возможно, это сервер популярной онлайн-игры или высокочастотной торговой системы). Каждая миллисекунда на счету, и вы знаете, что правильный сборщик мусора с точно настроенными параметрами может обеспечить максимальную отзывчивость и минимальные "зависания". Вам нужно не просто выбрать бригаду, но и дать им четкие инструкции, чтобы они работали эффективно и быстро.

Ваша задача: написать полную команду запуска Java-программы, которая будет использовать особый сборщик мусора G1 GC, известный своей эффективностью. Кроме того, вам нужно будет строго ограничить максимальный размер памяти, которую приложение может использовать (до 1 гигабайта), и установить желаемый максимум для времени, в течение которого сборщик мусора может "задерживать" работу вашего приложения (например, не более 150 миллисекунд). В комментарии к вашему коду, который может быть просто пустой программой, поясните назначение каждого флага в вашей команде запуска. Это позволит другим инженерам понять ваши настройки без лишних вопросов.

#### Требования:
1. В команде запуска должен быть указан флаг -XX:+UseG1GC для выбора сборщика мусора G1 Garbage Collector.
2. В команде запуска должен быть установлен флаг -Xmx1g, который ограничивает максимальный размер кучи Java до 1 гигабайта.
3. В команде запуска должен быть задан флаг -XX:MaxGCPauseMillis=150 для ограничения максимальной продолжительности паузы сборщика мусора до 150 миллисекунд.
4. В исходном коде программы должен быть комментарий, в котором объясняется назначение каждого из используемых JVM-флагов запуска.
5. В ответе должна быть приведена полная команда запуска Java-программы с использованием указанных параметров JVM.
