
### Симуляция нагрузки на сервер облачных вычислений ☁️

Вы — системный администратор, который отслеживает производительность крупного облачного сервиса. Чтобы понять, как система справляется с нагрузкой, вы запускаете 20000 имитированных клиентских запросов, каждый из которых требует обработки в течение 2 секунд. После активации всех этих запросов, вы хотите отслеживать пульс вашей системы в реальном времени.

Каждую секунду, в течение 3 последовательных интервалов, ваша программа должна выводить на экран два важных показателя: общее количество активно работающих потоков (как будто бы вы смотрите на текущее число занятых процессоров) и, отдельно, количество всех существующих виртуальных потоков (чтобы понять, сколько легких задач ожидает своей очереди или находится в процессе).

После завершения всех измерений и обработки всех запросов, убедитесь, что ваша программа дождалась полного завершения всех клиентских задач.

#### Требования:
1. Все 20 000 клиентских запросов должны быть обработаны с помощью виртуальных потоков (Virtual Threads), а не обычных потоков.
2. Каждый клиентский запрос должен выполняться в отдельном виртуальном потоке и занимать ровно две секунды (например, с помощью Thread.sleep(2000)).
3. Все 20 000 виртуальных потоков должны быть запущены практически одновременно, до начала измерения показателей.
4. В течение трех секунд после запуска всех задач программа должна каждую секунду (три раза подряд) выводить на экран: - текущее количество активно работающих потоков (живых потоков в системе), - текущее количество всех существующих виртуальных потоков.
5. Программа должна корректно дождаться завершения всех 20 000 виртуальных потоков перед своим завершением (например, с помощью join()/Future.get() или аналогичного механизма).
6. Вывод статистики должен происходить строго раз в секунду, три раза подряд, после запуска всех задач.
7. Для подсчета количества виртуальных потоков необходимо использовать корректные средства (например, Thread API или ManagementFactory) и отделять виртуальные потоки от обычных.
