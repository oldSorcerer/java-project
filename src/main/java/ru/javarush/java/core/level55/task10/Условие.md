
### Панель Пользователя: Быстрый Доступ к Данным с Умным Управлением

Вы разрабатываете персонализированную панель управления для пользователей крупного онлайн-магазина. Когда пользователь входит в систему, вам нужно быстро получить его имя, а затем, используя это имя, мгновенно подгрузить список его недавних заказов. Все это должно происходить максимально эффективно, с контролем за временем выполнения и использованием выделенных ресурсов.

Сначала, организуйте свой собственный "пул рабочих" — ExecutorService с тремя потоками, чтобы асинхронные задачи выполнялись параллельно и не мешали основной работе приложения.

Теперь, создайте две ключевые функции: getUserAsync(int id), которая, имитируя запрос к базе данных, вернет имя пользователя через 1 секунду, и getUserOrdersAsync(String username), которая, используя полученное имя, найдет список всех заказов этого пользователя, но уже с задержкой в 2 секунды.

В вашей основной программе, сначала получите имя пользователя по его идентификатору. Как только имя будет известно, немедленно, без промедления, приступайте к получению списка заказов этого пользователя, используя thenCompose, и обязательно поручите эту работу вашим специально выделенным рабочим из ExecutorService.

И вот критичный момент: получение списка заказов не должно занимать слишком много времени. Добавьте к этой операции тайм-аут в 1 секунду с помощью orTimeout. Если заказы не будут получены за это время, система должна элегантно вернуть пустой список, чтобы пользователь не ждал.

В конечном итоге, после завершения всей этой цепочки асинхронных операций, выведите на экран полный список заказов пользователя. И не забудьте, по завершении всех задач, корректно завершить работу вашего пула потоков (ExecutorService), чтобы освободить системные ресурсы.

#### Требования:
1. В программе должен быть создан экземпляр ExecutorService с тремя потоками для выполнения асинхронных задач.
2. Метод getUserAsync(int id) должен возвращать CompletableFuture<String>, имитировать задержку 1 секунду и выполнять работу в ExecutorService.
3. Метод getUserOrdersAsync(String username) должен возвращать CompletableFuture<List<String>>, имитировать задержку 2 секунды и использовать ExecutorService.
4. После получения имени пользователя, получение заказов должно начинаться немедленно с помощью thenCompose, чтобы обеспечить последовательное выполнение.
5. Все асинхронные операции (и получение имени, и получение заказов) должны выполняться в созданном пользователем ExecutorService, а не в ForkJoinPool.commonPool().
6. К операции получения заказов пользователя должен быть применён orTimeout с лимитом в 1 секунду.
7. Если заказы не получены за 1 секунду, программа должна обработать TimeoutException и вернуть пустой список заказов, не выбрасывая ошибку наружу.
8. После завершения всей цепочки операций, программа должна вывести на экран итоговый список заказов пользователя.
9. После завершения всех задач ExecutorService должен быть корректно завершён с помощью shutdown().
