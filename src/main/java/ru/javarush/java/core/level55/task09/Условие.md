
### Запрос Статуса: Мгновенный Ответ с Резервом

Представьте, что вы управляете критически важным сервисом, который постоянно ждет обновлений статуса от удаленных систем. Эти обновления могут приходить с задержкой, но ваш сервис не может ждать вечно; он должен реагировать быстро, имея запасной план.

Создайте асинхронную операцию (CompletableFuture), которая имитирует получение статуса "Completed" от удаленной системы, но с реалистичной задержкой в 2 секунды. Однако ваш сервис не терпит промедления! Установите на эту операцию строгий тайм-аут в 1 секунду с помощью completeOnTimeout. Если статус не придет вовремя, система должна автоматически принять "Default value" как актуальное состояние. Как только процесс завершится, будь то по факту получения данных или по срабатыванию тайм-аута, выведите на экран итоговый статус, который был определен.

#### Требования:
1. Задача должна использовать класс CompletableFuture для создания асинхронной операции.
2. Асинхронная операция должна имитировать получение статуса "Completed" с задержкой в 2 секунды.
3. На асинхронную операцию должен быть установлен тайм-аут в 1 секунду с помощью метода completeOnTimeout.
4. Если результат не получен в течение 1 секунды, итоговым значением должен стать "Default value".
5. После завершения операции (по факту или по тайм-ауту) итоговый статус должен быть выведен на экран.
