
### Финансовый Страж: Управление Рисками в Асинхронных Транзакциях

Вы — главный архитектор финансовой системы будущего, где каждая транзакция обрабатывается с максимальной эффективностью. Иногда, в критических операциях, могут возникать непредвиденные сбои. Ваша задача — создать механизм, который сможет предвидеть и мягко обработать такие ошибки, не допуская краха всей системы.

Сымитируйте критическую асинхронную транзакцию с помощью CompletableFuture.supplyAsync, где произойдет намеренное деление на ноль (например, 10 / 0), что, как известно, вызовет ArithmeticException. Ваш финансовый страж должен быть готов к этому: сразу после запуска этой рискованной операции выведите на экран "Старт задачи!". Затем, используйте специальный "аварийный протокол" (exceptionally), чтобы в случае возникновения такого исключения, вместо фатальной ошибки, система вернула бы осмысленную строку "Ошибка вычисления".

Вне зависимости от того, успешно ли прошла транзакция или случился сбой, используйте систему отчетности (thenAccept), чтобы на экран было выведено либо корректное числовое значение, либо предупреждающая строка "Ошибка вычисления".

#### Требования:
1. В задаче должен быть создан асинхронный CompletableFuture с помощью метода supplyAsync.
2. Внутри supplyAsync необходимо выполнить деление на ноль (например, 10 / 0), чтобы гарантированно получить ArithmeticException.
3. Сразу после запуска CompletableFuture (до обработки результата) на экран должно быть выведено сообщение "Старт задачи!".
4. Для обработки возможного исключения в CompletableFuture должен быть использован метод exceptionally, который возвращает строку "Ошибка вычисления" при возникновении ошибки.
5. Должен быть использован thenAccept для вывода результата вычисления (либо числа, либо строки "Ошибка вычисления") на экран.
6. Вне зависимости от исхода операции (успех или ошибка) результат должен быть выведен только через thenAccept.
