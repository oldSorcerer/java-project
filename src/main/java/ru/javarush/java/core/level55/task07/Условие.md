
### Отчеты Сенсора: Гибкая Обработка Данных с Восстановлением

Вы программируете критически важную систему контроля качества, которая получает данные от уникального сенсора. Сенсор обычно выдает идеальные показания, но иногда, из-за внешних помех, может произойти сбой. Ваша система должна быть к этому готова, умея восстанавливаться после ошибок и всегда выдавать обработанный результат.

Сымитируйте работу этого сенсора: он запускается асинхронно и в нормальных условиях возвращает число 10. Но будьте осторожны: если удача повернется к вам спиной (например, Math.random() окажется больше 0.5), сенсор выдаст серьезное исключение ArithmeticException. Вам нужно добавить специальный обработчик ошибок (exceptionally), который в случае такого сбоя будет возвращать безопасное значение 0, чтобы система не рухнула. После того, как исходное показание (или восстановленное значение) будет получено, независимо от того, был ли сбой, используйте следующий этап обработки (thenApply), чтобы удвоить это значение. В конце концов, выведите на экран окончательный, обработанный и проверенный результат работы вашего сенсора.

#### Требования:
1. Задача должна быть реализована с использованием CompletableFuture для асинхронного запуска сенсора.
2. Асинхронная задача должна возвращать число 10, либо выбрасывать исключение ArithmeticException с вероятностью 50% (если Math.random() > 0.5).
3. В случае возникновения исключения в асинхронной задаче должен использоваться метод exceptionally, который возвращает безопасное значение 0.
4. После получения исходного или восстановленного значения должен применяться thenApply, который удваивает полученное число.
5. Программа должна вывести на экран итоговое значение, полученное после всех этапов обработки (удвоенное число или 0, если был сбой).
