
### Мониторинг Системы: Детальное Логирование Событий

Вы разрабатываете высоконадежную систему мониторинга, которая должна не только обрабатывать различные события, но и предоставлять подробные отчеты о каждом этапе, включая информацию о возможных ошибках.

Инициируйте асинхронную задачу, которая, к сожалению, намеренно завершится с ошибкой, выбросив IllegalArgumentException с сообщением "Некорректный аргумент".

Теперь ваша задача — внедрить умный обработчик (handle), который сможет распознать эту ошибку: если она присутствует, он должен вернуть информативную строку вроде "Ошибка: <текст ошибки>", а если бы ошибки не было (в другом сценарии), то строку "Всё хорошо".

Кроме того, вам необходим всеобъемлющий механизм для завершающей отчетности (whenComplete), который выведет на экран полный лог, включающий как итоговое значение, так и флаг, указывающий, была ли ошибка, например: "Результат: <значение>, Ошибка: <есть ли ошибка>".

После всех этих этапов, наконец, покажите миру финальный результат выполнения вашей задачи.

#### Требования:
1. Необходимо создать CompletableFuture, который завершится с выбрасыванием IllegalArgumentException с сообщением "Некорректный аргумент".
2. Требуется использовать метод handle для обработки результата выполнения CompletableFuture. Если возникла ошибка, вернуть строку "Ошибка: <текст ошибки>", если ошибки нет — "Всё хорошо".
3. После обработки handle необходимо применить whenComplete для вывода на экран строки вида "Результат: <значение>, Ошибка: <есть ли ошибка>" (где <есть ли ошибка> — true/false или аналогичный индикатор).
4. В финале программа должна вывести итоговое значение, возвращённое обработчиком handle.
5. В задаче должны использоваться именно handle и whenComplete для обработки ошибок и логирования.
