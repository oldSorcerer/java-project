
### Диспетчер

Вы – диспетчер полетов для высоконагруженного Java-приложения, которое работает как настоящий космический корабль, выполняя критически важные операции. Чтобы убедиться, что оно летает стабильно и эффективно, вам нужно включить "черный ящик" – Java Flight Recorder – и потом расшифровать записанные им данные. Ваша миссия – провести полноценный "полетный тест".

Запустите свое Java-приложение, добавив специальные параметры для JVM: -XX:StartFlightRecording=filename=test.jfr,duration=30s,settings=profile. Это настроит запись "полетных данных" в файл test.jfr в течение 30 секунд, используя стандартный профиль производительности. После того как приложение завершит свою работу и запись будет закончена, откройте файл test.jfr в JDK Mission Control (JMC) – вашем аналоге центра управления полетами.

Внутри JMC найдите жизненно важную статистику: сколько раз за эти 30 секунд система занималась "уборкой мусора" (GC Events), чтобы поддерживать чистоту в своем "двигателе"? Укажите точное количество этих событий.

#### Требования:
1. Приложение должно быть запущено с параметром -XX:StartFlightRecording=filename=test.jfr,duration=30s,settings=profile для активации Java Flight Recorder.
2. Параметр duration должен быть установлен на 30 секунд, а settings должен быть profile.
3. В результате работы Flight Recorder должен быть создан файл с именем test.jfr.
4. Файл test.jfr должен быть открыт в программе JDK Mission Control (JMC) для анализа.
5. В JMC необходимо найти и определить количество событий Garbage Collection (GC Events), произошедших за период записи.
6. В ответе должно быть указано точное число событий GC, обнаруженных в файле test.jfr за 30 секунд записи.
