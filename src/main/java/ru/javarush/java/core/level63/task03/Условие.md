
### Архитектор

Представьте себя архитектором высоконагруженной финансовой системы, где каждая миллисекунда на счету и каждое событие должно быть зафиксировано с ювелирной точностью. Ваша цель — разработать идеальный шаблон для каждой записи в этом журнале, который будет служить основным источником информации для аудита и отладки.

Вам нужно определить строковый формат для логирования, который будет включать в себя дату и время события с максимальной детализацией (вплоть до миллисекунд, в формате "yyyy-MM-dd HH:mm:ss.SSS"). Каждое сообщение также должно ясно указывать на свой уровень важности (например, INFO или ERROR), имя класса или модуля, который его отправил, и, конечно же, сам текст сообщения.

Представьте, что итоговый результат должен выглядеть примерно так: "2024-06-16 18:42:07.123 INFO com.example.MainApp - Пользователь вошёл в систему".

#### Требования:
1. Формат лог-сообщения должен включать дату и время события с точностью до миллисекунд в виде "yyyy-MM-dd HH:mm:ss.SSS".
2. В формате лог-сообщения обязательно должен присутствовать уровень важности события, например, INFO, ERROR, WARN и т.д.
3. Формат должен содержать имя класса или модуля, из которого поступило сообщение.
4. В формате обязательно должно быть место для текстового сообщения, отражающего суть события.
5. Элементы формата должны следовать в следующем порядке: дата и время, уровень, имя класса/модуля, дефис, текст сообщения. Разделителем между именем класса и сообщением должен быть символ " - ".
6. Итоговый формат строки должен соответствовать примеру: "2024-06-16 18:42:07.123 INFO com.example.MainApp - Пользователь вошёл в систему".
