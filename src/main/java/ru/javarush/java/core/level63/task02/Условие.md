
### Новый разработчик

Как новый разработчик в команде, вы внедряете самую современную систему логирования для нового супермаркета, который работает полностью онлайн. Ваша команда только что интегрировала SLF4J как основной механизм для всех сообщений в системе. Ваш первый шаг – убедиться, что система работает как часы и записывает все необходимые данные.

Вам необходимо создать класс и настроить его для вывода трех типов сообщений. Сначала выведите информационное сообщение: "Привет, логирование через SLF4J!", чтобы подтвердить успешную инициализацию. Затем имитируйте небольшую проблему, отправив предупреждение: "Внимание: возможна проблема". Наконец, покажите, как система реагирует на серьезный сбой, отправив сообщение об ошибке: "Ошибка через SLF4J". Для этого обязательно получите логгер через LoggerFactory.getLogger и убедитесь, что все сообщения корректно отображаются в консоли.

#### Требования:
1. Класс должен использовать библиотеку SLF4J для логирования сообщений.
2. Логгер должен быть получен через вызов LoggerFactory.getLogger с указанием текущего класса.
3. Класс должен вывести в лог информационное сообщение "Привет, логирование через SLF4J!" с помощью метода info().
4. Класс должен вывести в лог предупреждение "Внимание: возможна проблема" с помощью метода warn().
5. Класс должен вывести в лог сообщение об ошибке "Ошибка через SLF4J" с помощью метода error().
6. Все три сообщения должны быть корректно отображены в консоли при запуске программы.
