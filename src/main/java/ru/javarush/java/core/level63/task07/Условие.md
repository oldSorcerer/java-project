
### Детектив

Вы – детективы по производительности в крупной IT-компании. Ваша задача – найти "виновника" медленной работы приложения, которое со временем начинает потреблять слишком много памяти, как будто в ней копится что-то ненужное. Почувствуйте себя настоящим следователем, которому предстоит разгадать тайну утечки памяти!

Для начала вам нужно запустить Java-приложение, которое имитирует такую ситуацию, постоянно создавая объекты в цикле, подобно примеру MemoryLeakDemo из вашей лекции. Затем, используя мощный инструмент VisualVM, подключитесь к работающему процессу. Вам нужно перейти на вкладку "Monitor" и нажать заветную кнопку "Heap Dump". Это как сделать моментальный снимок всего содержимого памяти приложения. После этого глубокого “вскрытия” вам предстоит определить, какой конкретно тип объектов безжалостно пожирает всю память, чтобы вы могли его "обезвредить". Укажите этот тип объекта как результат своего расследования.

#### Требования:
1. Необходимо создать и запустить Java-приложение, в котором реализована искусственная утечка памяти путем постоянного создания объектов в цикле.
2. Требуется использовать инструмент VisualVM для подключения к процессу запущенного Java-приложения.
3. Необходимо на вкладке "Monitor" в VisualVM сделать снимок памяти приложения с помощью функции "Heap Dump".
4. Требуется проанализировать полученный дамп памяти и определить, какой тип объектов занимает наибольший объем в куче.
5. В качестве результата необходимо указать тип объекта, который потребляет наибольшее количество памяти и, вероятно, является причиной утечки.
