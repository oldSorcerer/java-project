
### Эко-активист

Вы – эко-активист в мире кода, и ваша миссия – предотвратить "загрязнение" памяти ненужными, избыточными объектами. Вы подозреваете, что одно из ваших приложений слишком расточительно расходует ресурсы, создавая одни и те же объекты снова и снова. Почувствуйте, что вы – детектив, расследующий утечку ресурсов!

Ваша задача — написать программу, которая намеренно будет "мусорить" в памяти. Для этого организуйте цикл, который будет выполняться 500000 раз. В каждой итерации цикла создавайте новую строку с помощью new String("hello") (обратите внимание на new – это важно!) и тут же добавляйте эту строку в ArrayList.

После того как программа завершит этот "мусорный" цикл, запустите ее и затем, с помощью VisualVM, загляните внутрь памяти. Вам нужно "отловить" и посчитать, сколько таких избыточных объектов типа String накопилось в вашей "корзине памяти" после завершения этой операции. Если вы все сделали правильно, вы увидите огромное количество уникальных объектов типа String, подтверждая ваше подозрение о расточительности.

#### Требования:
1. Программа должна содержать цикл, который выполняется ровно 500 000 раз.
2. В каждой итерации цикла должна создаваться новая строка с помощью выражения new String("hello").
3. Каждая созданная строка должна быть добавлена в коллекцию ArrayList.
4. В результате выполнения программы в памяти должно оказаться 500 000 различных объектов типа String, несмотря на одинаковое содержимое.
5. Программа должна быть реализована так, чтобы после завершения её работы можно было с помощью инструментов профилирования (например, VisualVM) проанализировать количество созданных объектов String.
