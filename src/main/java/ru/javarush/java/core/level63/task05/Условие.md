
### Игровой сервер

Представьте, что ваш игровой сервер кипит от активности: тысячи игроков одновременно сражаются, строят и торгуют в вашей новой массовой многопользовательской онлайн-игре. Каждое действие игрока должно быть зафиксировано, чтобы можно было отследить баги, читеров или просто проанализировать игровой процесс.

Ваша задача — убедиться, что даже при одновременной записи, каждое сообщение от каждого игрока остается четким и неповрежденным в общем логе, демонстрируя мощь потокобезопасного логирования. Создайте класс, в котором два имитированных игровых потока одновременно будут записывать сообщения в лог, используя SLF4J (или любую другую библиотеку, такую как Log4j, Logback, java.util.logging).

Каждый поток должен отправить в лог по три уникальных сообщения. Проверьте, что в итоговом логе сообщения от разных потоков не "перемешиваются" внутри себя, а отображаются полностью и корректно, сохраняя свою целостность.

#### Требования:
1. Для логирования сообщений должен использоваться потокобезопасный логгер, например, SLF4J, Log4j, Logback или java.util.logging.
2. В программе должен быть создан класс, в котором запускаются два отдельных потока для имитации действий игроков.
3. Каждый поток должен записать в лог по три уникальных сообщения, отличающихся друг от друга.
4. Оба потока должны выполнять запись в лог одновременно, чтобы проверить потокобезопасность логирования.
5. Все сообщения в итоговом логе должны быть полностью записаны без перемешивания частей сообщений между потоками.
6. Для записи сообщений должен быть использован официальный API выбранной библиотеки логирования (например, logger.info, logger.debug и т.д.).
7. В итоговом логе не должно быть частичных или повреждённых сообщений, каждое сообщение должно быть целиком и без артефактов многопоточности.
