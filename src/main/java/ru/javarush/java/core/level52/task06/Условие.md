
### Реестр глобальных настроек приложения ⚙️

Вы создаёте критически важный реестр конфигураций для сложного распределённого приложения. Многочисленные компоненты системы будут часто читать эти настройки, но лишь изредка администратор должен иметь возможность их обновлять. Для обеспечения высокой производительности при чтении и сохранения целостности данных при записи, вы решили использовать ReadWriteLock.

Создайте класс ConfigurationRegistry. Внутри него должно быть приватное поле Map<String, String> для хранения настроек и поле типа ReadWriteLock для управления доступом.

Реализуйте метод setSetting(String key, String value): этот метод должен получить writeLock перед тем, как добавить или обновить элемент в карте, а затем освободить его, гарантируя, что никакие параллельные чтения не помешают процессу записи.

Также реализуйте метод getSetting(String key): этот метод должен получить readLock перед извлечением значения из карты (позволяя множеству читателей работать одновременно) и затем освободить его.

В вашей основной программе, моделирующей работу приложения, создайте объект ConfigurationRegistry. Сымитируйте, как администратор устанавливает новую конфигурацию: вызовите setSetting("apiEndpoint", "https://api.myapp.com/v1"). Затем сымитируйте, как часть приложения считывает эту настройку: вызовите getSetting("apiEndpoint") и выведите полученный URL на экран, подтверждая, что конфигурация доступна и корректно считана.

#### Требования:
1. В классе ConfigurationRegistry должно быть приватное поле типа ReadWriteLock для управления конкурентным доступом к настройкам.
2. В классе ConfigurationRegistry должно быть приватное поле Map<String, String> для хранения конфигурационных параметров.
3. Метод setSetting(String key, String value) должен получать writeLock перед изменением Map и освобождать его после завершения операции.
4. Метод getSetting(String key) должен получать readLock перед чтением значения из Map и освобождать его после завершения операции.
5. Ни один другой доступ к Map не должен производиться вне блоков, защищённых соответствующим локом (writeLock или readLock).
6. В основной программе должен быть создан объект ConfigurationRegistry, в который с помощью setSetting добавляется пара ("apiEndpoint", "https://api.myapp.com/v1").
7. После установки настройки в основной программе должен быть вызван getSetting("apiEndpoint"), а результат выведен на экран.
