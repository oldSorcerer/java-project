
### Отправка мгновенного сообщения: асинхронная запись с надёжным закрытием ✉️

Представьте, что вы создаете компонент для мгновенной отправки коротких сообщений. Когда пользователь нажимает "Отправить", сообщение должно быть немедленно записано в файл, не заставляя его ждать. При этом критически важно, чтобы после записи все ресурсы были аккуратно освобождены.

Ваша задача — написать программу, которая асинхронно записывает строку "Hello, async world!" в совершенно новый файл. После того как асинхронная операция записи будет полностью завершена, вам необходимо гарантировать корректное закрытие канала. Вы можете достичь этого, используя удобную конструкцию try-with-resources, которая автоматически позаботится о закрытии, или явно вызвав метод close() для канала после того, как убедитесь, что запись завершена. В итоге, когда ваша программа завершит работу, этот новый файл должен содержать отправленную вами строку.

#### Требования:
1. Для записи необходимо использовать асинхронный канал, такой как AsynchronousFileChannel.
2. Файл для записи должен быть создан заново, а не перезаписывать или дописывать в существующий.
3. Строка "Hello, async world!" должна быть записана в файл с использованием асинхронной операции записи.
4. После завершения асинхронной операции записи канал должен быть корректно закрыт — либо через try-with-resources, либо явным вызовом close().
5. Закрытие канала должно происходить только после того, как асинхронная операция записи полностью завершена.
6. В результате работы программы созданный файл должен содержать ровно строку "Hello, async world!".
