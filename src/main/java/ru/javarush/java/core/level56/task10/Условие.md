
### Синхронизация запуска системы: ожидание загрузки конфигурации ⚙️

Вы разрабатываете модуль для критически важной системы, которая перед запуском требует загрузки части своей конфигурации из файла. Эта операция чтения выполняется асинхронно, чтобы не блокировать другие части инициализации. Однако, дальнейшие шаги системы не могут начаться, пока загрузка конфигурации не будет полностью завершена (успешно или с ошибкой).

Ваша задача — реализовать программу, которая асинхронно читает первые 100 байт из файла (предположим, это "config.ini"). Для того чтобы основная программа знала, когда операция чтения завершена, вам следует использовать механизм CountDownLatch. Как только чтение (или попытка чтения) из файла будет завершена, будь то успех или ошибка, программа должна вывести на экран сообщение "Операция завершена" и только после этого позволить основной нити продолжить или завершить своё выполнение. Это гарантирует, что следующая фаза инициализации не начнется, пока эта ключевая операция не будет завершена.

#### Требования:
1. Программа должна читать первые 100 байт из файла "config.ini" асинхронно, не блокируя основной поток.
2. Для координации завершения асинхронной операции и основного потока должен использоваться CountDownLatch.
3. CountDownLatch должен уменьшаться (countDown) как при успешном завершении чтения, так и при возникновении ошибки.
4. После завершения (или неудачи) асинхронного чтения должно быть выведено сообщение "Операция завершена".
5. Основная программа не должна завершаться или переходить к следующей фазе, пока latch не будет отпущен (await).
6. Необходимо обеспечить, чтобы из файла читалось не более 100 байт, даже если файл больше.
7. В случае возникновения ошибок при чтении файла программа не должна зависать и должна корректно отпустить latch.
