
### Операция "Критический Срок": Обезвреживание Киберугрозы ⏱️

Вы — руководитель оперативной группы по кибербезопасности, и вашей команде поручена миссия по обезвреживанию опасной вирусной угрозы. Эта операция состоит из трёх последовательных этапов: "Сканирование угроз", "Изоляция системы" и "Очистка данных". Время критически ограничено, и каждая операция должна быть завершена до общего, строго установленного дедлайна, иначе вся миссия будет провалена.

Реализуйте программный механизм, который будет отслеживать прогресс вашей команды. Ваша функция должна принять в качестве параметра точное время missionDeadline, к которому вся операция должна быть завершена. Каждый из трёх этапов займёт определённое время: сканирование — 500 миллисекунд, изоляция — 800 миллисекунд, и очистка — 900 миллисекунд. Перед началом каждого следующего этапа система должна проверить, достаточно ли времени осталось до missionDeadline.

Если в какой-то момент окажется, что оставшегося времени недостаточно для выполнения следующего этапа, система должна немедленно выдать сообщение об ошибке "Не уложился в дедлайн", сигнализируя о провале миссии. Если же ваша команда справилась со всеми этапами в срок, система должна поздравить вас с успехом, выведя на экран "Все операции выполнены вовремя". Запустите этот сценарий, установив missionDeadline на 2 секунды от текущего момента, чтобы проверить, сможет ли команда уложиться в такой жесткий срок.

#### Требования:
1. Перед началом каждого этапа операции программа должна проверять, достаточно ли времени осталось до значения missionDeadline для успешного завершения этапа.
2. Длительность этапов должна быть строго следующей: сканирование — 500 миллисекунд, изоляция — 800 миллисекунд, очистка — 900 миллисекунд.
3. Если перед началом любого этапа времени до missionDeadline недостаточно для его выполнения, программа должна немедленно вывести сообщение "Не уложился в дедлайн" и завершить выполнение.
4. Если все три этапа были выполнены в срок, программа должна вывести сообщение "Все операции выполнены вовремя".
5. Функция должна принимать параметр missionDeadline — абсолютное время, до которого должны быть завершены все этапы.
6. Сценарий должен быть запущен с missionDeadline, установленным на 2 секунды от текущего момента.
