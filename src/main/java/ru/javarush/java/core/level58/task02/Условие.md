
### Поиск Быстрейшего Сервера: Система "Сверхзвуковая Доставка Данных" ⚡

Вы разрабатываете ультрасовременную систему для доставки критически важных данных, где каждая миллисекунда на счету. Ваша цель — получить информацию от любого из трёх доступных серверов, который ответит первым. Вам абсолютно неважно, какой именно сервер пришлёт данные, главное, чтобы это произошло как можно скорее. Как только хоть один сервер присылает ответ, все остальные ожидания должны быть немедленно прекращены, чтобы не тратить драгоценное время.

Напишите программу, которая моделирует этот сценарий. У вас есть три виртуальных сервера, каждый из которых потенциально может отправить ответ: первый присылает "Сервер_А", второй — "Сервер_Б", и третий — "Сервер_В". Используйте механизм, который позволит запустить все три "запроса" одновременно и как только получите первый успешный отклик, тут же отобразить его на экране. Например, если "Сервер_А" окажется самым проворным, вы должны увидеть "Сервер_А".

#### Требования:
1. Программа должна запускать три задачи (запроса к серверам) параллельно, чтобы минимизировать время ожидания.
2. Каждая задача должна возвращать уникальное строковое значение: "Сервер_А", "Сервер_Б" или "Сервер_В" в зависимости от виртуального сервера.
3. Программа должна получить результат от того сервера, который ответит первым, и использовать только этот результат.
4. После получения первого ответа программа должна немедленно прекратить выполнение (отменить) остальные ожидающие задачи.
5. Программа должна вывести на экран строку, полученную от самого быстрого сервера.
