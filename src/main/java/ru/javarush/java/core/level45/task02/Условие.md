
### Расширение цифровой библиотеки и проверка совместимости

Представьте, что вы — хранитель огромной цифровой библиотеки, и ваша система умеет сохранять информацию о книгах в файлы. На первом этапе, когда система только запускалась, для каждой книги вы сохраняли только её название и автора.

Для этого создайте класс Book с полями bookTitle типа String и bookAuthor также типа String. Обязательно сделайте этот класс сериализуемым, чтобы его можно было сохранять и восстанавливать. Очень важно сразу же присвоить этому классу уникальный идентификатор версии, добавив статическое финальное поле serialVersionUID со значением 1L.

Теперь, имитируя первую версию вашей системы, создайте один или несколько объектов Book (например, "Война и мир" Льва Толстого) и сериализуйте их в файл.

Прошло время, и ваша библиотека развивается. Возникла необходимость добавить новую, очень важную информацию о книгах — год их публикации. Однако у вас уже есть тысячи сохранённых файлов со старой структурой! Ваша задача — обновить класс Book, добавив в него новое поле publicationYear типа int, но при этом НЕ ИЗМЕНЯТЬ значение serialVersionUID. Это ключевой момент для проверки совместимости.

После внесения изменений в класс, попробуйте десериализовать те самые объекты Book, которые вы сохранили в самом начале, ещё до добавления поля publicationYear. Выведите на экран значения всех полей десериализованного объекта: bookTitle, bookAuthor и publicationYear. Вы должны увидеть, что старые данные bookTitle и bookAuthor успешно восстановились, а новое поле publicationYear, которое отсутствовало в старом файле, получило своё значение по умолчанию, то есть 0. Это продемонстрирует, как Java обрабатывает изменения в структуре классов при сериализации, сохраняя совместимость со старыми версиями.

#### Требования:
1. Класс Book должен быть объявлен сериализуемым и реализовывать интерфейс Serializable.
2. В первой версии класса Book должны быть объявлены два поля: bookTitle и bookAuthor, оба типа String.
3. В классе Book должно быть объявлено статическое финальное поле serialVersionUID со значением 1L.
4. В первой версии программы должен быть создан хотя бы один объект Book, который сериализуется в файл.
5. После сериализации, в классе Book должно быть добавлено новое поле publicationYear типа int, при этом значение serialVersionUID не изменяется.
6. После добавления поля publicationYear, необходимо десериализовать ранее сохранённый объект Book из файла.
7. После десериализации, значения полей bookTitle и bookAuthor должны быть восстановлены из файла.
8. После десериализации, поле publicationYear должно иметь значение по умолчанию для типа int (то есть 0), так как это поле отсутствовало в старой версии сериализованных данных.
9. После десериализации, значения всех трёх полей (bookTitle, bookAuthor, publicationYear) должны быть выведены на экран.
